## 3. App Router structure

Root layout: [app/layout.tsx](app/layout.tsx)
- Sets `<html lang="en">`, mounts `<Navbar />` above `{children}`, wraps content in `<main className="p-8">`, and mounts Vercel `<Analytics />` at the end of `<body>`.
- `metadata`: title `"B. A."`, description `"Personal portfolio of B. Anderson"`.

### `/` — [app/page.tsx](app/page.tsx)
Client component (`"use client"`). Two-column layout on desktop (`md:flex-row`, `md:h-screen`), stacked on mobile:
- **Left column** — profile photo (`/profile.jpeg`), "About Me" blurb, and three action buttons: Download Resume (`/ResumeBennettAnderson.pdf`, direct download), LinkedIn, GitHub. Button gradient colors branch on a local `dark` boolean.
- **Right column** (`id="scroll-panel"`, independently scrollable, scrollbar hidden via injected `<style>` tag) — in order:
  1. **Featured Projects** — hardcoded `featuredProjects` array (2 entries: "Portfolio Website", "Resume, Redesigned!"), each linking to `/projects`.
  2. **GitHub Repositories** — live-fetched from `https://api.github.com/users/B-M-Anderson/repos` in a `useEffect`, filtered to `!repo.fork && repo.owner.login === "B-M-Anderson"`. Renders each repo with a thumbnail resolved from `/thumbnails/${repo.name.replace(/\s+/g, "")}.png`, falling back to `/thumbnails/default.png` via the `<Image>` `onError` handler.
  3. **Skills Section** — renders `<SkillsSection />` (see below).
  4. **Cat Photo Album** — hardcoded `catPhotos` array of 6 images from `public/cats/`.
  5. A footer note about desktop dark-mode being the optimized viewing experience.

Dark-mode detection on this page uses `window.matchMedia("(prefers-color-scheme: dark)")` directly — it does **not** observe the `<html>` class the way `Navbar`, `SkillsSection`, and `ProjectsPage` do (see §6 for why this matters).

### `/projects` — [app/projects/page.tsx](app/projects/page.tsx)
Client component. Renders the `projects` array (hardcoded, see §5) as an accordion list:
- A sort dropdown (Date new→old / old→new, Completion high→low / low→high) re-sorts a **copy** of the array (`sortedProjects` state); the underlying `projects` const is never mutated.
- Each row is a collapsible card. Header shows title, thumbnail (`thumbnailUrl`), a media badge (see §5), and the date. Expanding reveals: optional preview image (`imageUrl`), status bars, full description (`whitespace-pre-line`), skill badges (important skills sorted first, highlighted in purple), two `ProgressBar` components (Completion, Estimated AI Usage), and a "View on GitHub" link.
- `ProgressBar` is a local component with its own dark-mode `useEffect`/`MutationObserver`, a gradient track (purple for completion, green for anything with "ai" in the label), and a hover tooltip showing the exact percentage.
- A footer note about limited mobile functionality.

### `/contact` — [app/contact/page.tsx](app/contact/page.tsx)
Client component. Animated gradient header ("Contact 📬"), then an email link (`mailto:bennetta32.30@gmail.com`), a phone number (`(815) 821-9604`, not a `tel:` link), and a LinkedIn link. No form — direct contact info only.

## 4. Components

### [app/components/Navbar.tsx](app/components/Navbar.tsx)
- Fixed, full-width, `z-50`. Shows full name on `lg+`, "BA" abbreviation below that, plus a "company security software" disclaimer hidden below `sm`.
- **Dark mode**: on mount, reads `prefers-color-scheme` via `matchMedia` and seeds `dark` state; a `useEffect` toggles `dark`/`light` classes on `document.documentElement` whenever `dark` changes. The toggle button (`Moon`/`Sun` icon) flips `dark` directly — this is the **only** source of truth for the `<html>` class, which is why other components sync via `MutationObserver` on that class rather than re-deriving from `matchMedia` themselves.
- **Scroll-hide behavior**: on mobile, tracks `window.scrollY`; on desktop, tracks the scroll position of `#scroll-panel` (the right column on the homepage — on `/projects` and `/contact`, which don't have that element, `scrollTop` reads `0` via optional chaining, so the navbar won't hide on those routes). Scrolling down hides the bar (`opacity-0 -translate-y-full`), scrolling up or being near the top (`y < 20`) shows it.

### [app/components/SkillsSection.tsx](app/components/SkillsSection.tsx)
- Renders 4 accordion categories — Computational Tools, MicroBiology, Data & Analysis, Soft Skills — each a hardcoded `{ name, strong }[]` list. `strong: true` skills sort first and get a green badge; others get gray.
- Syncs its own `dark` state via `MutationObserver` on `document.documentElement`'s `class` attribute (the correct pattern — matches how `ProjectsPage`/`ProgressBar` do it).
- `mounted` state gate prevents the ▲/▼ arrow from rendering until after hydration (avoids SSR/client mismatch).

## 5. The `Project` type (`app/projects/page.tsx`)

```ts
type Project = {
  title: string;
  description: string;
  githubUrl?: string;
  date: string;
  skills: string[];
  importantSkills?: string[];
  media?: "photo" | "video" | "both" | "none";
  aiUsage?: number;
  completion?: number;
  thumbnailUrl?: string;
  imageUrl?: "/Previews/BennettAndersonResume1.png";
  wip?: boolean;
  terminated?: boolean;
  completed?: boolean;
  ongoing?: boolean;
  shelved?: boolean;
};
```

| Field | Drives |
|---|---|
| `title` | Card header `<h2>` |
| `description` | Full text shown when expanded (`whitespace-pre-line` preserves manual line breaks) |
| `githubUrl` | "View on GitHub" link at the bottom of the expanded card (omitted entirely if absent) |
| `date` | Shown under the title; also the sort key for the two date-based sort options (parsed via `new Date(project.date)`, so it must stay in a JS-parseable format like `"November 22, 2025"`) |
| `skills` | Badge list; sorted so `importantSkills` come first |
| `importantSkills` | Subset of `skills` that render in purple instead of gray |
| `media` | Drives the small badge next to the title: `"photo"` → Camera icon "Photo Demo", `"video"` → Video icon "Video Demo", `"both"` → both icons "Photo & Video", `"none"`/undefined → no badge |
| `aiUsage` | Green-gradient `ProgressBar` labeled "Estimated AI Usage" (0–100) |
| `completion` | Purple-gradient `ProgressBar` labeled "Completion" (0–100) |
| `thumbnailUrl` | Small square thumbnail next to the title in the collapsed header |
| `imageUrl` | Large preview image shown at the top of the expanded card. **Note:** typed as a string literal union of exactly one value (`"/Previews/BennettAndersonResume1.png"`) rather than `string` — so today only the Resume Refresh project can use this field without a type error; adding a second project's preview image requires widening this type first (see `CLAUDE.md`) |
| `wip` / `terminated` / `completed` / `ongoing` / `shelved` | Each renders its own full-width colored status bar (orange/red/green/blue/yellow respectively) with a hover tooltip explaining what the status means, only inside the expanded view. These are independent booleans — nothing prevents setting more than one true simultaneously (current data never does) |

## 7. Styling

- [app/globals.css](app/globals.css): imports Tailwind v4 (`@import "tailwindcss";`), defines CSS custom properties for light/dark backgrounds and foregrounds, and applies them via `html.light` / `html.dark` selectors (the class Navbar toggles). Base `body` reset with `font-family: Arial, Helvetica, sans-serif`.
- [postcss.config.mjs](postcss.config.mjs): single plugin, `@tailwindcss/postcss` — Tailwind v4's PostCSS integration, no separate `tailwind.config.js`.
- **Dark mode mechanism**: not the standard `next-themes`/media-only approach. `Navbar` is the single source of truth — it seeds from system preference once, then toggles `dark`/`light` classes on `<html>` in response to the user's manual click. Every other component that needs to know the current theme (`SkillsSection`, `ProjectsPage`, `ProgressBar`) independently attaches a `MutationObserver` to `document.documentElement` watching the `class` attribute. `app/page.tsx` (the homepage) is the one exception — it re-derives `dark` from `matchMedia` only, so it will not reflect a manual toggle (see suggestion in the wrap-up conversation).

## 9. Analytics & deployment

- **Analytics**: `@vercel/analytics`'s `<Analytics />` component is mounted once, in [app/layout.tsx](app/layout.tsx), so it covers every route automatically.
- **Hosting**: Vercel, auto-deploying every push to `main` (per the operator, this is genuinely wired up — treat all `main` pushes as production deploys).
- **Domain**: custom domain purchased through Squarespace, pointed at Vercel (mentioned in both the homepage copy and the `/projects` "Bennett-Anderson.com" entry description).
- **GitHub integration**: the homepage's "GitHub Repositories" section is a live client-side fetch to the public GitHub REST API (`api.github.com/users/B-M-Anderson/repos`, unauthenticated, so subject to GitHub's ~60 req/hr per-IP rate limit for anonymous requests) — no server component, no caching layer, no API route. Filters out forks and repos not owned directly by `B-M-Anderson` (e.g. org repos would be excluded).
